- 问题： epub.js左右翻页的时候，想要提取页面文本，rendition.currentLocation()是最接近的方法，
但是它返回的CFI定位，级别只到<p>，没有到char级别，导致提取的文本都是整段落的，会提取当前页面同一个段落的没有显示再页面的那些文字
方法： 借助AI帮助，分析项目 https://github.com/futurepress/epub.js 和文档 http://epubjs.org/documentation/0.3/，
找到原因是 rendition.mananger.mapping.splitTextNodeIntoRanges() 这个函数中，默认使用的分隔符是空格，
所以如果是英文的话，能正常获取当前页面显示的文本，但是如果是中文的话，就没有空格可以分隔，所以只能获取到整段文本，
而且也没有适合中文的分隔符作为参数给它，所以要重写这个函数并且hook，不使用分隔符，而是每个字符分割，效率降低了。
另外注意 rendition.manager.mapping 在 rendition.manager.updateLayout() 中会被重重建，
所以也要 hook updateLayout，重建后重新hook splitTextNodeIntoRanges。
hook 时需要 rendition 准备好，我是在第一次display().then()中进行的。
方法： hook updateLayout，在原来的 updateLayout 执行完成后，替换 manager.mapping 的3个函数为自己的版本
findTextStartRangeByChar,findTextEndRangeByChar,splitTextNodeIntoRanges，重写这三个函数，使用每个字符分割而不是空格分割

- 学习 Promise：
const myPromise = new Promise(func)
const myPromise = new Promise((resolve, reject) => {})
const myPromise = new Promise((a, b) => {})
Promise(func)必需接受一个函数作为参数func，func需要接收两个参数，两个参数都是 Promise 给函数，两个函数参数又分别可以接收一个参数
语义上两个参数习惯取名resolve, reject，
刚创建时，Promise 处于 pending 状态，直到 Promise 给的两个函数参数其中一个执行，
如果执行的是第一个，则 Promise 跳转到 fulfilled 状态，
如果执行的是第二个，则 Promise 跳转到 rejected 状态，
跳转状态后，Promise 会执行 then() 中的函数，then(resolved,rejected) 接收两个函数参数， 这两个函数需要接收一个参数
如果是 resolve(p1) 跳转，则执行 then 中的第一个参数函数 resolved(p1)，并且把参数传给它，这里的 p1 是一样的
如果是 rejected(p2) 跳转，则执行 then 中的第二个参数函数 rejected(p2)，并且把参数传给它，这里的 p2 是一样的
当你创建了一个 Promise 实例后，Promise(func) 中的 func 立即就开始同步执行，是同步执行，
需要等 func 执行完毕才能返回 Promise，所以 func 中通常要启动一些异步代码以便快速返回

例子：
const myPromise = new Promise((resolve, reject) => { 这里调用 resolve 或者 reject 触发状态跳转，或者暴露它们给其他地方调用，调用异步代码，尽快结束当前函数 })
myPromise.then(func1(p1){},func2(p2){});
只要在某处调用 resolve('called resolve')， p1 = 'called resolve'， func1(p1) 就会被执行。
reject 同理。

- 问题： epub滚动模式下（不是分页模式），点击导航栏，跳转到章节，执行的是 rendition.display(href)，
页面滚动到章节末尾，或者中间，就是不滚动到章节开头。分页模式就没有问题。
方法： 在ai的帮助下，尝试了很多方法，还是没有解决。
后来发现可能是渲染的异步问题，而且发现如果再次点击同样的章节，就能定位到开头了。
所以直接在代码中，连续两次display，即 rendition.display(href).then(rendition.display(href).then(...))
这样虽然会有闪烁，好歹正确定位了。先使用这样的方法吧。
分页模式的话，虽然测试正常，但是担心不是一般情况，所以上面的方法也用到了分页模式中，没有区分是滚动还是分页，都使用两次display。

- 问题： epub刚开始初始化以后，首次调用display显示上次阅读位置。旋转屏幕时，页面重建，不是全屏时，很正常。
但是，当全屏状态下旋转屏幕时，display().then(..这里一直没有触发..)。
原因是，当全屏状态下，旋转屏幕以后，会触发resize。epub.js内部resize与display如果同时运行，会导致递归调用进入display进入死循环。
如果不是全屏的话，就不会触发resize，就不会有问题。我怀疑是旋转屏幕后，状态栏重新隐藏需要时间。
如果是手动点击的话，此时并没有在运行display，也不会有问题。
方法： 在 attached 事件触发后，延迟500ms再调用display，保佑500ms内resize完成。如果没有resize，也能正确显示。
还有，在 resize 事件触发后，防抖后立即启动display。 自定义标记变量，保证首次display只执行一次，而且不影响其他地方的display。
